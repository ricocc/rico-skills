# Rico Resource Article - 详细使用指南

资源推荐文章生成器，通过多轮交互创建高质量的资源推荐类文章。

---

## 核心特性

- **智能分步生成**：搜集候选 → 用户确认 → 生成标题 → 生成文章
- **广泛领域支持**：工具、资源、组件库、插件、网站、框架、服务等
- **多种生成模式**：智能分步、一步到位、基于大纲生成
- **专业格式规范**：完整 Front Matter + 标准文章结构
- **双语支持**：中英文输入/输出
- **可选截图**：自动为推荐网站添加截图

---

## 触发规则

### ✅ 会触发技能

必须同时包含：**生成动作** + **文章类型**

```
生成一篇关于 shadcn/ui 的推荐文章
写一个工具推荐文章
创建资源列表文章
推荐 8 个组件库并生成文章
Write a recommendation article about React hooks
```

### ❌ 不会触发技能

只有询问或对话，没有生成文章的意图：

```
推荐 8 个组件库
有哪些好的工具
What are some React hooks
```

## 文章结构

### Front Matter（元数据）

```yaml
---
title: 选择的标题
publishDate: 2026-03-11
read: 5
description: English subtitle describing the article
tags:
  - 中文标签
  - English Tag
img: "/assets/blog/cover-filename.jpg"
img_alt: "图片描述"
---
```

### 开头段落（3-4 段）

- 背景：行业现状和问题
- 价值：主题的核心意义
- 引出：文章要推荐的内容

### 正文（推荐项目）

```markdown
### 1. 项目名称

- 地址：[https://example.com](https://example.com)
- 简介：核心定位、主要功能、解决的问题...
- 优势：独特价值、适用场景...

    ![project-name.jpg](project-name.jpg)
```

### 结尾总结

- 升华主题的小标题（## 二级标题）
- 3-4 段总结，结合趋势给出建议

## 写作风格要求

### 避免 AI 写作常见问题

| 问题 | ❌ 错误示例 | ✅ 正确示例 |
|------|-----------|-----------|
| 过度排比 | "它具有高效性、灵活性、可扩展性" | "它高效且灵活，易于扩展" |
| 空泛描述 | "这个工具非常强大，功能非常丰富" | "包含 200+ 组件，覆盖表单、导航等场景" |
| 三段式 | "不仅...而且...还..." | 使用自然多样的句式 |
| 否定式排比 | "它不需要复杂配置，不需要额外依赖" | "配置简单，零依赖" |

### 写作原则

- 专业但不晦涩
- 每个推荐有明确使用场景
- 原创内容，不照搬官方介绍
- 不添加作者署名

## 可选截图功能

文章生成后可选择自动添加截图。

### 支持的服务

| 服务 | 免费额度 | 速度 | API Key |
|------|---------|------|---------|
| **ScreenshotOne** | 100 次/月 | 2-5 秒 | 需要 |
| **urlscan.io** | 5,000 次/天 | 10-30 秒 | 需要 |

### 配置方法

```bash
# 创建配置目录
mkdir -p .rico-skills

# 复制示例配置
cp .env.example .rico-skills/.env

# 编辑 .rico-skills/.env
SCREENSHOTONE_ACCESS_KEY=your_key
URLSCAN_API_KEY=your_key
SCREENSHOT_DEFAULT_SERVICE=screenshotone
SCREENSHOT_DEFAULT_SIZE=1600x900
```

### 获取 API Key

- **ScreenshotOne**: https://screenshotone.com/（免费注册，100 次/月）
- **urlscan.io**: https://urlscan.io/（登录后 Settings → API Keys）

### 截图尺寸

| 尺寸 | 宽高比 | 适用场景 |
|------|--------|----------|
| 1920×1080 | 16:9 | 全高清 |
| 1600×900 | 16:9 | **默认** - 博客文章 |
| 1440×900 | 16:10 | Mac 风格 |
| 1200×800 | 3:2 | 平板设备 |

可在请求时指定："用 1920×1080 截图"

## 常见问题

### 技能没有触发？

确保同时包含：
1. **生成动作**：生成、写、创建、帮我写
2. **文章类型**：推荐文章、资源列表文章

### 生成的文章太长/太短？

可以在确认候选列表时说明：
- "每项简介控制在 2 句话"
- "每个推荐详细一些，3-4 句话"

### 想要调整风格？

生成后可以用自然语言调整：
- "语气再专业一点"
- "加一些使用建议"
- "结尾再加一段总结"
